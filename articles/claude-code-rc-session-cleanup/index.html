Claude Code のリモートセッション、溜まりませんか

← orange-wks
Claude Code2026-04-16

Claude Code の Remote Control を使っていると、claude.ai のサイドバーに切断済みのセッションが溜まっていく。

1つ2つなら気にならないが、数十件になると邪魔だ。公式には一括削除の機能がない。UIから1件ずつ消すしかない。

ブックマークレットで解決する

ブラウザのブックマークバーに以下を登録しておけば、claude.ai を開いた状態でワンクリックで掃除できる。

セットアップ:

  1. ブックマークバーを右クリック →「ページを追加」
  2. 名前: Claude Sessions 掃除（何でもいい）
  3. URL 欄に以下を貼る:
javascript:void(async()=>{const h={"anthropic-beta":"ccr-byoc-2025-07-29","anthropic-client-feature":"ccr","anthropic-client-platform":"web_claude_ai","anthropic-version":"2023-06-01","content-type":"application/json"};let a=[],u="/v1/sessions";while(u){const r=await(await fetch(u,{headers:h})).json();(r.data||r).forEach(s=>a.push(s));u=r.has_more&&r.last_id?"/v1/sessions?after_id="+r.last_id:null}const t=a.filter(s=>s.status!=="active");if(!t.length){alert("削除対象なし (全"+a.length+"件がactive)");return}if(!confirm("全"+a.length+"件中 "+t.length+"件の非activeセッションを削除しますか？")){return}let d=0,f=0;for(let i=0;i<t.length;i+=10){await Promise.all(t.slice(i,i+10).map(s=>fetch("/v1/sessions/"+s.id,{method:"DELETE",headers:h,body:"{}"}).then(r=>r.ok?d++:f++).catch(()=>f++)))}alert("完了！ 削除="+d+" 失敗="+f);location.reload()})()

使い方は、claude.ai を開いてブックマークをクリックするだけ。確認ダイアログが出るので件数を見て OK を押す。active なセッションはフィルタで除外されるので、進行中のものは消えない。

何をやっているか

ブラウザがすでに持っているログインセッションを使って、内部 API を叩いている。

  1. GET /v1/sessions でセッション一覧を取得（ページネーション対応）
  2. status !== "active" のものだけフィルタ
  3. DELETE /v1/sessions/{id} を10件ずつバッチ実行

外部にリクエストは飛ばない。claude.ai のドメイン内で完結する。

制約

  • PC のブラウザでのみ動く（iPhone の Claude アプリでは DevTools もブックマークレットも使えない）
  • 非公式 API なので、将来仕様が変わる可能性はある
  • CLI 側に claude sessions delete のようなコマンドが追加されれば不要になる（Feature Request は出ている）
