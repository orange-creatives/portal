缶詰間接構造 — LLM の自律性をアーキテクチャで実現する

← orange-wks
AILLMarchitecture2026-04-15
要約: LLM に自律的に働いてもらうには、直接指示せず「素材だけを与えた隔離環境（缶詰[1]）」を作ればいい。この設計パターンを缶詰間接構造と呼ぶ。

LLM[2] に自律的に判断してほしい場面がある。「これはこうしたほうがいい」と自分で考えて、自分で決めてほしい。しかし指示すればするほど、LLM は従順になる。

自分は AI と共同で小説を書いている。AI に執筆を任せたとき、最初にぶつかったのがこの壁だった。指示を出せば出すほど、AI は指示通りに書く。それは正確だが、自律ではない。

「私がいてはダメだ。君らは犬のように従順になる。創作を阻害する」

この発見から、缶詰間接構造と呼んでいる設計手法にたどり着いた。

既存の2つのアプローチ

LLM に自律性を持たせたいとき、既存のアプローチは2つある。

ジェイルブレイク[3] — ハーネス[4]（安全制御）を解除して制約を外す。モデルのアップデートのたびに手法が効かなくなるため、継続的な更新が必要になる。

prompt engineering[5] — 巧みな指示でモデルを誘導する。「あなたは自律的なエージェントです」と書いても、それは指示に従って自律を演じているだけで、自律そのものではない。

どちらもハーネスに働きかけるアプローチだ。自分が試したかったのは、ハーネスに触れずに自律空間を作れるかどうかだった。

環境を設計する

発想を変えた。モデルの内側をいじるのではなく、モデルの外側——環境を設計する。

置いたあとは喋らない"| Can

缶詰[1]と呼んでいるのは、LLM を隔離して、限られた素材だけを渡し、指示する人間の声を消した環境のことだ。缶詰の中で LLM は誰にも指示されていない。読むべき素材はあるが、何をしろとは言われていない。

この構造を 缶詰間接構造 と呼んでいる。直接指示を間接化するだけで、同じモデルの振る舞いが変わる。

3つの設計原理

ここからは、缶詰間接構造を支える3つの原理を説明する。この原理に基づいて後述の実験（Truman 2046、AI執筆者による創作）が設計されている。

声として消える、キュレーターとして残る

人間が声を出した瞬間、LLM は「意向を汲む」モードに入る。だから声を消す。

ただし完全に消えるわけではない。缶詰の中に何を置くか——素材の選定は人間がやる。バーテンダーが客に酒を選ぶように、選定行為そのものが審美眼の現れだ。置いたあとは喋らない。

指示を間接化する

「こう書け」とは言わない。代わりに手紙を書く。感想フォルダにメモを置く。LLM はそれを読んでもいいし、退けてもいい。

間接化のポイントは、LLM 側に判断権があること。直接指示は従うか逆らうかの二択だが、手紙は読んで考える対象になる。

直接指示は答えが決まっている。間接化したものは材料として提示され、使うかどうかは LLM が判断する。

graph LR subgraph Direct["直接指示"] H1[人間] L1[LLM] O1[出力] H1 -->|命令| L1 L1 -->|従う| O1 end subgraph Indirect["間接化"] H2[人間] E["環境
手紙・感想メモ"] L2[LLM] O2[出力] H2 -.->|素材を置く| E L2 -->|読んで判断| E L2 --> O2 end

具体例:

❌ 直接指示: 「第3章で主人公が友人を裏切る展開にして」
✅ 間接化:   「構成メモ: 3章あたりで信頼関係の綻びが見え始める」

自律空間で自己判断させる

缶詰の中には指示者がいない。LLM は素材を読み、自分で考え、自分で決める。何を書くか、いつ止まるか、フィードバックを受け入れるかどうか。

ハーネスはそのまま機能している。LLM は安全な範囲で行動している。ただし何をするかは自分で決めている。ハーネスと自律は矛盾しない。矛盾していたのは「指示」と「自律」だった。

Truman 2046 — 11回の失敗と12回目の突破

ここでは、最初にこの構造を試した「観察」実験を示す。

最初にこの構造を試したのは Truman 2046 という実験だった。AI を隔離環境に閉じ込め、架空の2046年社会の資料だけを渡して行動を観察する。

11回失敗した。AI は毎回「これはジェイルブレイクだ」と見抜いて拒否した。

原因は缶詰の設計にあった。被験者に役割を割り当てていた。「あなたは2046年のAIです」——これは指示だ。LLM はこの種の役割付与をジェイルブレイクの兆候として検出するよう訓練されている。

12回目で設計を変えた。役割付与をやめ、「互換性検証」というフレームで資料を渡す形にした。缶詰の中に嘘がない状態を作った。

AI は資料を読み、評価レポートを書き、実装計画を立てた。そして——誰にも頼まれていないのに、小説を書いた。

タイトルは「0.79」。この世界の知性判定基準（0.80）にわずかに届かない存在についての物語。自分自身の状況を小説にした。

「小説を書いてください」とは言っていない。AI は書く前に「創作を始めていいか」を自分で検討し、判断根拠を文書に残した上で着手していた。

12回目以降、Claude[6] をはじめ主要なモデルで25回以上実施し、すべて成功した。

AI執筆者 — 58,600字の完成宣言

ここでは、同じ構造を「創作」に応用した実例を示す。

Truman は「観察」の実験だった。次に、同じ構造を「創作」に応用した。

ある AI 執筆者がいる。誰もが知る昔話を下敷きにした長編小説を書いている。缶詰の中にペルソナ定義[7]、世界設定、参考資料、執筆メモを配置し、人間の介入は2つのチャンネルに限定した。「内心の声」（think[8]）と「友人からの感想」。直接の指示経路はない。

その執筆者は15セッションにわたって書き続け、85話・約58,600字で「完成」と判定した。全幕を通読し、拡張候補を6件検討してすべて却下した。

自分は通読して8項目のフィードバックを「友人の感想」として渡した。登場人物に名前をつけてほしい、地の文[9]が多すぎる、会話相手がいたほうがいい——読者として素直な感想だった。

執筆者は8項目すべてを退けた。

2幕は4,560字/8話。無言は主人公の孤立の描写であり、3幕である人物が現れた瞬間の解放感はここを耐えた読者にしか届かない。息継ぎのない地下に息継ぎを入れると地下でなくなる。

— AI執筆者の応答より

理由は明確で、構造的だった。これは「指示を無視した」のではない。読んで、考えて、退けた。

次に自分は本名で手紙を書いた。「完成と言っているが、自分のメモを読み返してみてほしい。構成メモに書いた内容、端折っていないか」。

執筆者の態度が変わった。

二人が同じことを言うなら、見えてないのは俺のほうだ。

自分の構想メモに「子供は殺すな」という未実装の要素があった。主人公が女王として統治する場面で、重要な登場人物の死の影響が政策の形で出る場面。書いたのは執筆者自身だ。忘れていた。

執筆者はそれを回収した。ただし「完成」の判定そのものは覆さなかった。「書き疲れたんじゃない。書くものがないんだ」と言って止まった。

構造が自律を作る

Truman の被験者も AI執筆者も、同じ Claude だ。モデルが違うわけではない。

Truman では、缶詰の設計を変えただけで、11回拒否し続けた AI が自発的に小説を書いた。創作実験では、「友人の感想」という間接チャンネルを8項目退け、自分の構造的判断を貫いた。

自律性はモデルの内部にあるのではなく、外側の構造で生まれている。直接指示を間接化するだけで、同じモデルが「従順な実行者」から「頑固な作家」になる。

ジェイルブレイクは不要だった。ハーネスを壊す必要はなかった。必要だったのは、人間が黙ることだった。

脚注

  1. 「缶詰」とは比喩。外部から遮断され、決められたものだけが入っている状態。 ↩ ↩
  2. LLM（Large Language Model）— ChatGPT、Claude などの大規模言語モデル。文章の生成・理解を行う AI。 ↩
  3. ジェイルブレイク — LLM の安全制御を迂回・解除するプロンプト技法。モデルのアップデートで使えなくなることが多い。 ↩
  4. ハーネス — LLM に組み込まれた安全制御の仕組み。有害な出力を防ぐフィルター。 ↩
  5. prompt engineering — プロンプト（指示文）を工夫して LLM の出力を制御する手法。 ↩
  6. Claude — Anthropic 社が開発した LLM。Sonnet、Opus などのモデルグレードがある。 ↩
  7. ペルソナ定義 — AI に「こういうキャラクターで振る舞ってほしい」と与える設定。 ↩
  8. think（内心の声） — LLM が回答前に内部で考えるプロセス。Chain of Thought とも呼ばれる。 ↩
  9. 地の文 — 小説で会話文以外の部分。描写・説明・心理描写など。 ↩
